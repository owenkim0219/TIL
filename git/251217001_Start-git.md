## How to use git?

1. Set configuration

```shell
$ git config --global user.name "{github username}"
$ git config --global user.email "{github mail addr}"
$ git config --global core.editor "vim"
$ git config --global core.pager "cat"
```

2. Create remote repository on github and clone it.

`$ git clone {repo addr}`

3. Set configuration

- .gitignore
    - https://www.toptal.com/developers/gitignore/
    - OS, Language, Editors, Frameworks
- README.md

4. Do work!!

```shell
$ touch main.py
$ vi main.py
# After work,
$ git status
$ git add main.py
$ git commit
$ git push origin main
```

## Branch

분기점으로 부터 코드가 독립적으로 존재할 수 있도록 도와주는 git의 모델

```python
# main.py(main branch)
print('hello')

# main.py(develop branch)
word = 'hello'
for c in word:
    print(c, end='')
```

### How to use branch?

```shell
# 브랜치 리스트 확인
$ git branch
$ git branch -r
$ git branch -a

# 브랜치 생성
$ git branch fizz

# 브랜치 변경
$ git branch fizz

# 브랜치 병합(fizz > main, on main branch)
$ git merge fizz
```

### Merge conflict

- 서로 다른 브랜치에서 한 파일의 같은 라인에서 다른 변화가 있을 경우 발생
- `merge` 혹은 `rebase` 로 해결 가능
- 충돌난 상황을 해결하는 전혀 다른 솔루션이 나와도 상관없음
- 구분선 (`<<<<<<<========>>>>>>>>`)은 작업 종료 시 꼭 제거하기

1. `merge`로 해결하기

```shell
$ git merge buzz
# 충돌 발생 상황 확인
$ git status
$ vi fizzbuzz.py
# 충돌난 파일들 해결 후,
$ git add fizzbuzz.py
$ git commit
```
